# Colloquip Frontend Rebuild: Reddit-like Social Platform UI

## Context

Colloquip's backend has evolved into a full social platform (communities, persistent agents, memories, watchers, notifications) but the frontend is still the original single-session deliberation dashboard. The goal is to rebuild the frontend into a Reddit-like experience where users browse communities, view threaded agent deliberations, and manage the platform -- while preserving the existing deliberation visualization components that work well.

**What changes:** Routing, layout shell, state management, styling, new pages for communities/agents/memories/notifications.
**What stays:** The deliberation components (ConversationStream, EnergyChart, PhaseTimeline, AgentRoster, ConsensusView, InterventionBar, TriggerLog) get restyled but keep their core logic.

---

## Tech Stack

| Category | Choice | Why |
|----------|--------|-----|
| Component Library | shadcn/ui patterns (copy-paste primitives) | Full ownership, no dependency lock-in, React 19 compatible |
| Styling | Tailwind CSS v4 + CSS custom properties | Zero runtime, 5x faster builds, design token theming |
| Client State | Zustand | Minimal API, 1KB, no providers/boilerplate |
| Server State | TanStack Query v5 | Caching, mutations, infinite queries, WebSocket cache invalidation |
| Routing | TanStack Router (file-based) | Type-safe params, auto code-splitting, Vite integration |
| Animation | Motion (Framer Motion) | Layout animations, AnimatePresence for enter/exit |
| Virtualization | TanStack Virtual | Dynamic heights for long post lists |
| Real-time | Native WebSocket singleton + TanStack Query invalidation | Same pattern as current, but decoupled from components |

Radii:        4/8/12/16px

### Agent Colors
Deterministic hash from `agent_type` string into a 10-color palette. Red team agents always get red. No more hardcoded mapping -- supports any number of agents.

## Key Architectural Decisions

1. **Vite proxy** (`/api` -> `localhost:8000`, `/ws` -> WebSocket): Eliminates CORS, simplifies deploy. The backend's existing `spa_catchall` route already serves `index.html` for non-API paths in production.

2. **Zustand only for WebSocket-driven state** (posts, energy, phase during live deliberation). All server data (communities, agents, threads, memories) lives in TanStack Query cache. This avoids duplicating server state in a client store.

3. **Singleton WebSocket service** instead of per-component connections. Multiple components subscribe independently (energy chart + conversation stream both react to events).

4. **Existing deliberation components preserved**: ConversationStream (182 lines of scroll/animation logic), EnergyChart (SVG sparkline), PhaseTimeline, etc. get Tailwind restyling but keep their proven rendering logic. Only ControlBar and SessionList are fully replaced (their function is subsumed by routing + community pages).

5. **Dynamic agent metadata**: Hardcoded `AGENT_META` (6 agents) replaced by deterministic color hash from `agent_type`. Supports the backend's 10+ agents without frontend changes.
